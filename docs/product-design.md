## 3. 产品定位

这是一个轻量、离线优先的桌面截图工具，核心能力是：

1. 快速框选屏幕区域。
2. 在选区上进行简单标注。
3. 复制或保存截图。
4. 将截图钉在屏幕上作为临时参考。
5. 使用操作系统自带 OCR 识别选区文字。
6. 通过托盘和可选的全局快捷键快速启动截图。

产品不替换 Windows 自带截图或开始菜单，不修改系统默认行为，以独立桌面程序运行。

### 3.1 目标用户

- 日常办公、开发和资料整理用户。
- 需要在文档、网页、代码和聊天窗口之间快速截取内容的用户。
- 课堂演示、远程协作和临时对照场景中的桌面用户。

### 3.2 产品原则

- 核心流程短，不要求登录，不依赖云端。
- 默认设置可直接使用，高级设置保持克制。
- 截图、标注、复制、保存和钉住必须稳定优先。
- 设置界面采用紧凑桌面工具风格，不做卡片化或营销页式布局。
- 免费功能不捆绑大型 OCR 模型。

## 5. 核心用户流程

### 5.1 区域截图

1. 用户从主窗口、托盘菜单或全局快捷键启动区域截图。
2. 程序建立当前顶层窗口候选，并一次性抓取完整虚拟桌面快照。
3. 透明无边框输入画布覆盖所有显示器；待选区阶段不绘制桌面快照、暗层或窗口高亮，只将鼠标显示为十字光标，用户直接看到实时桌面。
4. 左键按下但尚未移动时画面保持不变；按住鼠标拖动超过阈值后一次性显示静态桌面快照和统一暗层，并在拖选区域重绘原始像素。
5. 用户单击可按窗口范围选取；按住鼠标拖动超过阈值时自动切换为自由框选。
6. 单击窗口或松开拖选时，程序从缓存快照裁剪选区，不再二次抓取屏幕。
7. 选区保持在原始屏幕位置和原始比例，不跳到左上角，不自动放大或缩小。
8. 截图完成后继续显示暗层，在选区原始位置绘制原图、边框、尺寸信息和工具条。
9. 用户可调整选区、标注、复制、OCR、钉住或取消；截图模式中单击鼠标右键立即取消截图。

### 5.2 标注与输出

截图工具条的目标顺序为：

`撤销｜重做｜画笔｜矩形｜箭头｜文字｜橡皮擦｜马赛克｜OCR｜钉住｜取消｜保存｜复制`

颜色和线宽作为标注工具的紧凑附加控件显示；文字工具复用三级尺寸控件。工具按钮只显示图标，鼠标悬停时按当前界面语言显示提示。复制是最常用的完成动作，固定放在工具条最右侧；保存使用独立按钮，不再通过动态主输出按钮切换行为。

OCR 使用未叠加标注的原始选区图像；复制、保存和钉住使用已经合成标注的图像。

文字输入使用 Slint 原生 `TextInput` 处理焦点、输入法和回车确认，最终输出由 Windows GDI 将文字合成到图像像素中。橡皮擦按拖动轨迹命中并删除标注，一次按下到松开的擦除作为一个可撤销步骤；马赛克按笔刷轨迹像素化底图，并与其他标注共同参与撤销、重做和最终输出。两者都不直接修改原始截图。

### 5.3 钉住

1. 用户点击“钉住”后，当前截图在原始选区位置创建无标题栏窗口。
2. 初始尺寸与原始截图物理尺寸一致，不自动缩图。
3. 窗口保持原始宽高比。
4. 用户可拖动、滚轮缩放、通过右键菜单调整窗口和图像。
5. 钉住窗口默认置顶，并保留一条清晰但克制的边框。

产品用语统一为“钉住”，不再使用“贴图”。

### 5.4 钉住右键菜单

在钉住图片上点击鼠标右键，应打开图片上下文菜单。当前“右键循环透明度”的行为应删除，透明度改为菜单中的明确选项。

首批实现菜单：

| 菜单项 | 行为 |
| --- | --- |
| 复制图像 | 复制当前图像内容，不包含窗口边框、阴影、透明度和显示缩放 |
| 图像另存为... | 按原始像素尺寸保存当前图像，格式遵循保存对话框选择 |
| OCR 识别 | 识别当前图像并将文字复制到剪贴板；系统 OCR 不可用时显示简短原因 |
| 窗口阴影 | 当前钉住窗口的勾选开关，初始值来自设置 |
| 始终置顶 | 当前钉住窗口的勾选开关，初始值来自设置 |
| 不透明度 | 子菜单：25%、50%、75%、90%、100% |
| 缩放 | 子菜单：25%、50%、75%、100%、125%、150%、200%、适合屏幕、恢复原始大小 |
| 替换图像 | 子菜单：从剪贴板、从文件... |
| 在文件夹中显示 | 仅当图像存在本地源文件或已经保存时显示 |
| 关闭 | 关闭当前钉住窗口，不影响其他钉住窗口 |

第二批增强菜单：

| 菜单项 | 行为 |
| --- | --- |
| 显示工具栏 | 在图片边缘显示完整共享截图工具条，提供撤销、重做、画笔、矩形、箭头、文字、橡皮擦、马赛克、颜色、尺寸、保存、复制、OCR 和关闭 |
| 图像处理 | 子菜单：向左旋转 90°、向右旋转 90°、水平翻转、垂直翻转 |

菜单规则：

- 菜单按“图像输出、窗口显示、图像调整、来源操作、关闭”分组，并使用分隔线。
- 菜单使用独立无边框置顶窗口，不受钉住图片边界裁剪，并按当前显示器工作区自动向左或向上展开。
- 当前生效的阴影、置顶、不透明度和缩放比例应显示勾选状态。
- “替换图像”保留当前窗口位置；默认按新图像原始比例调整窗口尺寸。
- 从剪贴板替换时，剪贴板没有可用图像应显示明确提示，不静默失败。
- 从文件替换只接受应用支持解码的图片格式。
- “在文件夹中显示”不能用于尚未保存的内存截图。
- 菜单底部可显示只读信息：`原始尺寸：宽 × 高`，不做多级尺寸菜单。

以下参考软件菜单不进入当前产品：

- “移动到分组”：需要钉住历史库和分组数据模型，超出当前轻量定位。
- “销毁”：当前没有持久化钉住历史，与“关闭”含义重复且容易误操作。
- 单独的“粘贴”和“替换为文件”：合并为“替换图像”子菜单，减少菜单长度。

## 6. 窗口与界面结构

### 6.1 单窗口单视图

主程序有且仅有一个 `MainWindow`。不设置首页、控制面板、设置页之间的视图切换，也不创建独立设置弹窗。

窗口内容直接从五个标签开始，不增加重复的产品标题或输出摘要：

`常规｜截图｜钉住｜快捷键｜关于`

底部操作区左侧保留“开始截图”，右侧保留“恢复默认”和“保存”。窗口最底部使用单行状态栏统一显示就绪、成功和错误消息，错误不再散落到各标签页。错误状态以红色 `×` 标识，其他状态以 `√` 标识；状态栏保持固定高度，长消息通过悬停提示查看完整内容，不弹出阻断工作流的消息框。

区域截图也可从托盘或用户配置的全局快捷键启动。启动时不自动最小化；关闭系统标题栏按钮等同于隐藏到托盘，不结束进程。

界面采用紧凑桌面工具样式：

- Windows 默认字体优先使用 Microsoft YaHei UI，正文约 12–13 px。
- 控件高度约 24 px，设置行高约 28 px。
- 内容边距约 10 px。
- 1 px 中性灰边框。
- 圆角控制在 0–2 px。
- 不使用嵌套卡片、大片阴影和大标题。
- 浅色、深色主题都应保持清晰对比。

页签中的设置控件采用未保存草稿模型：

1. 启动时从当前配置填充控件。
2. “保存”校验、持久化并立即应用设置。
3. “恢复默认”只修改当前标签页的草稿，不直接保存，也不影响其他标签页。
4. 外观模式和界面语言可即时预览；未保存时重启应用会恢复为已保存配置。

### 6.2 截图遮罩窗口

- 覆盖虚拟桌面。
- 无系统标题栏和窗口边框。
- 窗口背景透明；待选区阶段不绘制桌面快照、暗层或窗口高亮，只显示十字光标并透出实时桌面。
- 左键按下但尚未移动时画面保持不变；拖动超过阈值后才将启动截图时捕获的虚拟桌面快照作为固定底图。
- 静态底图出现时同步绘制统一半透明暗层，未选择区域保持稳定遮罩。
- 待选区阶段仍在内部识别光标下方的顶层可见窗口，单击后直接进入窗口选区。
- 单击光标下方的窗口进入窗口选区；拖动超过阈值进入自由选区。
- 智能窗口识别排除隐藏、最小化、DWM cloaked 和鼠标穿透窗口。
- 选区内部从缓存快照重绘原始像素，不依赖透明窗口挖洞。
- 完成选区时从缓存快照裁剪原图，避免遮罩、工具条和桌面后续变化进入截图。
- 截图模式中单击鼠标右键立即取消截图，并恢复启动截图前的主窗口状态。
- 选区边缘提供可调整大小的命中区域，但不显示突兀的黑色控制块。
- 工具条优先显示在选区下方，并与选区边框保留间距；空间不足时显示在选区上方。
- 提示信息不得被窗口或屏幕边缘裁切。

### 6.3 钉住窗口

- 无系统标题栏和关闭按钮。
- 无黑色侧边控制块。
- 1 px 边框。
- 默认置顶。
- 按原始宽高比缩放。
- 左键拖动。
- 滚轮缩放。
- 双击关闭。
- 右键打开图片上下文菜单。
- 透明度按设置值初始化。
- 截图和钉住工具条共用同一个 `CaptureToolbar` 组件，不维护两套视觉实现。
- 钉住工具栏保留完整标注能力，不使用只含复制、OCR 和关闭的精简模式。
- 钉住图像上的新标注在复制、保存和图像变换时合成到结果中；OCR 继续读取未叠加新标注的原图。

窗口显示状态与图像数据分离：

- 缩放只调整窗口尺寸，不重采样原图。
- 透明度、阴影、边框和置顶不写入图片。
- 旋转、翻转和替换属于图像数据操作，会影响后续复制、保存和 OCR。
- 每个钉住窗口可临时覆盖阴影、置顶、不透明度和缩放；新窗口仍使用全局设置默认值。

### 6.4 系统托盘

托盘保持常驻，菜单至少包含：

- 区域截图
- 显示控制面板
- 隐藏控制面板
- 退出

左键点击托盘图标直接恢复唯一主窗口。主窗口关闭按钮默认隐藏到托盘，不结束进程。

### 7.4 快捷键

| 设置项 | 默认值 | 状态 |
| --- | --- | --- |
| 区域截图 | `Ctrl+Alt+C` | 已实现 |

交互规则：

- 默认注册 `Ctrl+Alt+C`；注册失败时保留快捷键设置并显示明确错误。
- 输入框右侧提供清除按钮。
- 清空时立即注销当前运行中的快捷键，保存后持久化为空。
- 未设置快捷键时不显示状态图标。
- 注册成功时显示绿色有效图标。
- 注册失败时显示红色无效图标。
- 无效图标的悬停提示只使用简短原因：
  - `已被占用`
  - `组合键无效`
  - `系统拒绝注册`

不增加“截图并复制”“钉住”“显示全部钉住窗口”等额外快捷键。

### 7.5 关于

- Git 项目地址。
- 问题反馈地址。
- 版权信息。
- Slint 构建归属信息。

未来可增加：

- 检查更新。
- 登录和许可证状态。
- 专业版有效期。
- “升级专业版”入口。

## 8. OCR 设计

### 8.1 基础 OCR

基础 OCR 使用操作系统自带能力，不随应用打包大型模型，保持安装包轻量并支持离线使用。

| 平台 | OCR 实现 | 状态 |
| --- | --- | --- |
| Windows | `Windows.Media.Ocr` | 已实现 |
| macOS | Apple Vision | 规划中 |
| 其他平台 | 无 | 不做 |

基础 OCR 流程：

1. 用户完成区域选择。
2. 点击工具条 OCR 图标，提示为“识别选区文字”。
3. OCR 读取未叠加画笔、矩形和箭头的原始选区图像。
4. 识别成功后，将纯文本复制到系统剪贴板。
5. 主界面或截图状态区域显示结果。

统一状态文案：

- `已识别并复制文字`
- `未识别到文字`
- `缺少中文 OCR 语言包`
- `当前平台不支持系统 OCR`

OCR 使用独立设置页。用户可以明确选择兼容系统 OCR 或 Windows AI OCR；没有可用能力时不自动切换引擎。语言优先使用操作系统和用户已安装的语言包。

Windows 端保留 OCR 入口；系统能力或语言包不可用时显示统一的简短原因。非目标平台不显示无效入口。

### 8.2 Windows AI OCR

Windows 增强 OCR 使用 Windows App SDK 的 `Microsoft.Windows.AI.Imaging.TextRecognizer`。当前实现包括：

- 通过 `TextRecognizer.GetReadyState()` 检测 `Ready`、`NotReady`、`NotSupportedOnCurrentSystem` 和 `DisabledByUser`。Windows App SDK 1.8 不会把硬件、驱动和策略原因继续细分。
- 仅在返回 `NotReady` 时由用户点击“安装模型”，再调用 `EnsureReadyAsync()`。
- 使用 `RecognizedWord.MatchConfidence` 的 `0.0..=1.0` 返回值执行应用侧最低置信度过滤。
- 识别和能力检测均在同一程序的隔离子进程中执行，原生组件异常不会直接退出主界面。
- 使用 Windows App SDK WinMD 生成 Rust 绑定，不手写 COM 接口。

最低置信度是 GridStart 的输出过滤设置，不是 TextRecognizer 模型的输入参数。兼容系统 OCR 不返回词级置信度，因此该设置只对增强 OCR 生效。

Windows AI OCR 需要具有包身份的安装方式、`systemAIModels` 清单能力、对应 Windows App Runtime，以及受支持的 Windows 和硬件。仓库提供外部位置包身份模板；签名证书和商店 Publisher 在发布阶段配置。

当前开发环境尚未注册增强 OCR 所需的包身份和 Windows AI 运行组件，探针返回“组件或包身份不可用”；增强识别结果尚未完成受支持设备实测。应用必须显示明确的不支持原因，不能把接口存在视为设备可用。

### 8.3 后续专业 OCR

以下能力属于规划中的专业版，不计入当前开发承诺：

- 可选下载 PaddleOCR/ONNX 模型。
- 更高中文识别精度。
- 自动语言检测。
- 竖排文字和复杂版面。
- OCR 结果编辑。
- 批量 OCR。
- 表格、Markdown 等结构化导出。

专业模型应独立下载，不直接扩大基础安装包。

### 8.4 OCR 接口

共享业务层只依赖统一接口，不直接调用 Windows 或 macOS API：

```rust
pub trait OcrEngine {
    fn is_available(&self) -> bool;
    fn recognize(&self, image: &CapturedImage) -> Result<String, OcrFailure>;
}
```

具体类型和异步方式可在实现阶段调整，但平台能力不得直接散落在 Slint 回调中。

## 9. 配置结构

当前配置已经迁移到截图工具结构。旧版额外字段在读取时被忽略；用户首次保存后只写入下面的当前结构，不保留长期双写。

当前结构示例：

```json
{
  "appearance": "system",
  "language": "system",
  "launch_at_startup": false,
  "hotkey": null,
  "capture": {
    "format": "png",
    "jpeg_quality": 90,
    "save_directory": "",
    "filename_template": "Screenshot_{yyyy-MM-dd_HH-mm-ss}",
    "auto_save": false,
    "save_notification": true
  },
  "ocr": {
    "engine": "system",
    "minimum_confidence": 60
  },
  "pin": {
    "default_opacity": 100,
    "shadow": true,
    "always_on_top": true,
    "wheel_zoom": true,
    "zoom_step": 15,
    "double_click_close": true
  }
}
```

约束：

- 文件编码为 UTF-8。
- 写入采用临时文件加原子替换，避免中途失败破坏现有配置。
- 数值在反序列化后进行范围校验。
- 无效配置应返回明确错误并保留原文件，不静默覆盖。
- 配置保存路径在品牌名确定后需要统一迁移，避免长期使用 `GridStart`。
- `language` 可选 `system`、`chinese`、`english`；界面静态文案通过 Slint bundled translations 内嵌，不依赖外部语言包。

Slint 负责窗口、布局、控件状态和用户输入；Rust 负责截图会话、配置、输出、OCR、托盘命令和平台原生能力。

当前已经拆出的平台服务包括：

- Windows 屏幕截图。
- Windows 图像和文字剪贴板。
- Windows 顶层窗口识别与选区命中检测。
- `OcrEngine`
- 本地时间和原子文件替换。
- Windows 窗口控制。
- Windows 开机启动。
- Windows Shell 文件操作。

`app.rs` 只负责应用生命周期、主窗口、托盘、状态栏和快捷键事件；截图会话、设置事务和钉住窗口命令分别由独立控制器管理。纯图像旋转、翻转和标注渲染位于 `image.rs`，Windows 截图、窗口命中与剪贴板不再混入图像领域模型。

UI 不直接持有 Win32 句柄；跨线程 OCR 结果通过 Slint 事件循环回到 UI 线程更新，窗口恢复时主动请求重绘。配置和截图文件采用临时文件写入、刷新并原子替换，失败时保留原文件。

macOS 截图必须处理“屏幕录制”权限；权限缺失时给出明确提示，不返回空白截图。

## 13. 验收标准

### 13.1 截图

- 多显示器和不同 DPI 下，选区坐标正确。
- 快捷键启动后透明输入画布不绘制快照，不出现快照从窗口中心扩展、分块铺开或明显跳动。
- 待选区时只显示十字光标，不显示蒙版或窗口边界；单击可直接选择光标下方的窗口。
- 左键按下但尚未移动时画面保持不变；拖动超过阈值后才显示蒙版和自由选区。
- 按住左键拖动超过阈值后切换为自由框选，不误触窗口选择。
- 拖动时选区内部显示缓存快照原始像素，外层暗色遮罩均匀且无错乱横条。
- 遮罩、边框和工具条不进入最终截图。
- 主窗口可被截图，不因隐藏动画出现半透明画面。
- 截图模式中单击鼠标右键立即取消截图并恢复主窗口。
- 松开鼠标后选区不跳动、不放大、不改变比例。
- 移动或缩放选区时，内部截图内容应随边框实时更新，不迟滞、不在松开鼠标后跳变。
- 选区移动和缩放后，复制结果与可见内容一致。
- 标注只在按住左键期间绘制，松开后立即结束。
- 工具条不遮挡选区边框，屏幕边缘处不被裁切。
- 所有工具提示随界面语言切换中文或英文。

### 13.2 钉住

- 初始位置、尺寸和截图选区一致。
- 没有标题栏、黑色关闭按钮和侧边黑块。
- 缩放保持宽高比。
- 拖动、透明度、置顶和双击关闭行为符合设置。
- 可同时创建多个钉住窗口，并继续从托盘或主窗口发起新截图。
- 右键打开上下文菜单，不再直接循环透明度。
- 右键菜单作为独立置顶窗口完整显示，不能被钉住图片或屏幕工作区裁切。
- 右键菜单使用 Slint 原生 `ContextMenuArea` 和嵌套 `Menu`；二级菜单悬停展开，一级菜单保持显示，并由框架处理屏幕边缘避让。
- 截图和钉住工具条的按钮尺寸、间距、背景、边框及提示样式一致。
- 钉住工具栏可直接使用画笔、矩形、箭头、文字、橡皮擦、马赛克、撤销、重做、颜色、尺寸、保存和复制。
- 钉住图像复制和保存结果包含当前可见标注，OCR 不受标注干扰。
- 复制和另存为不包含窗口边框、阴影、透明度及显示缩放。
- “恢复原始大小”准确恢复截图像素尺寸。
- “适合屏幕”保持比例并完整显示，不超出当前显示器工作区。
- 替换图像后窗口位置不变，尺寸按新图像比例更新。
- 无本地源文件时不显示“在文件夹中显示”。

### 13.3 主窗口与托盘

- 用户主动隐藏后，可通过托盘左键和菜单恢复。
- 恢复窗口后完整重绘，无局部空白。
- 启动时不自动最小化。
- 关闭主窗口不退出进程，托盘“退出”才结束程序。

### 13.4 设置

- 主程序始终只有一个窗口和一个视图，不出现设置弹窗或首页/设置切换。
- “保存”后无需关闭窗口即可看到可热更新设置的效果。
- 外观三种模式均可切换，跟随系统能响应系统主题。
- 配置写入失败时保留旧配置并显示明确错误。
- 手动修改配置文件后，提示需要重启应用。
- 所有设置、截图、输出、OCR 和钉住操作错误统一显示在底部状态栏。

### 13.5 快捷键

- 默认不注册任何全局快捷键。
- 合法且未占用的组合键可触发区域截图。
- 清空后立即注销原快捷键。
- 失败图标和提示原因一致，不显示冗长错误。

### 13.6 OCR

- OCR 使用无标注的原始截图。
- 成功时复制纯文本。
- 无文字、缺少语言包和平台不支持时给出对应状态。
- OCR 处理不得阻塞 Slint UI 线程。
- 选择 Windows AI OCR 后不可用时显示具体状态，不静默切换到兼容 OCR。
- 最低置信度只过滤 Windows AI OCR 的 `MatchConfidence` 返回值。

### 13.7 工程质量

- `cargo fmt --check` 通过。
- `cargo test` 通过。
- `cargo clippy` 无新增高优先级问题。
- Release 构建通过。
- Windows 中文源码和文档严格使用 UTF-8。
- 浅色和深色主题下进行截图级视觉检查。

## 14. 明确不做

当前产品不包含：

- 替换系统开始菜单。
- 云同步、云笔记、相册和附件管理。
- 账号密码托管。
- Android、iOS、Web 和 Linux 客户端。
- 鼠标穿透。
- 钉住历史库、分组管理和“销毁”语义。
- 多组全局快捷键。
- 免费版内置大型 OCR 模型。
- 截图遮罩颜色、选区边框和锚点的高级外观设置。
- 未实现的会员、购买、登录和更新入口。

这些内容如需重新进入产品范围，必须单独评估，不直接混入截图工具主线。
