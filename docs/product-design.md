## 3. 产品定位

这是一个轻量、离线优先的桌面截图工具，核心能力是：

1. 快速框选屏幕区域。
2. 在选区上进行简单标注。
3. 复制或保存截图。
4. 将截图钉在屏幕上作为临时参考。
5. 使用操作系统自带 OCR 识别选区文字。
6. 通过托盘和可选的全局快捷键快速启动截图。

产品不替换 Windows 自带截图或开始菜单，不修改系统默认行为，以独立桌面程序运行。

### 3.1 目标用户

- 日常办公、开发和资料整理用户。
- 需要在文档、网页、代码和聊天窗口之间快速截取内容的用户。
- 课堂演示、远程协作和临时对照场景中的桌面用户。

### 3.2 产品原则

- 核心流程短，不要求登录，不依赖云端。
- 默认设置可直接使用，高级设置保持克制。
- 截图、标注、复制、保存和钉住必须稳定优先。
- 设置界面采用紧凑桌面工具风格，不做卡片化或营销页式布局。
- 免费功能不捆绑大型 OCR 模型。

## 5. 核心用户流程

### 5.1 区域截图

1. 用户从主窗口、托盘菜单或全局快捷键启动区域截图。
2. 主窗口临时隐藏，程序抓取虚拟桌面。
3. 全屏无边框遮罩覆盖所有显示器。
4. 用户按住鼠标左键拖出选区。
5. 选区保持在原始屏幕位置和原始比例，不跳到左上角，不自动放大或缩小。
6. 松开鼠标后，遮罩消失，仅保留选区图像、边框、尺寸信息和工具条。
7. 用户可调整选区、标注、复制、OCR、钉住或取消。

### 5.2 标注与输出

截图工具条的目标顺序为：

`撤销｜重做｜画笔｜矩形｜箭头｜复制｜OCR｜钉住｜取消`

颜色和线宽作为标注工具的紧凑附加控件显示。工具按钮只显示图标，鼠标悬停时显示中文提示。

OCR 使用未叠加标注的原始选区图像；复制、保存和钉住使用已经合成标注的图像。

上述“复制”是默认输出模式下的主输出按钮。当用户将“截图完成后”改为“保存”或“复制并保存”时，主输出按钮的图标和提示应同步变为对应动作，不能继续显示为单纯复制。

### 5.3 钉住

1. 用户点击“钉住”后，当前截图在原始选区位置创建无标题栏窗口。
2. 初始尺寸与原始截图物理尺寸一致，不自动缩图。
3. 窗口保持原始宽高比。
4. 用户可拖动、滚轮缩放、通过右键菜单调整窗口和图像。
5. 钉住窗口默认置顶，并保留一条清晰但克制的边框。

产品用语统一为“钉住”，不再使用“贴图”。

### 5.4 钉住右键菜单

在钉住图片上点击鼠标右键，应打开图片上下文菜单。当前“右键循环透明度”的行为应删除，透明度改为菜单中的明确选项。

首批实现菜单：

| 菜单项 | 行为 |
| --- | --- |
| 复制图像 | 复制当前图像内容，不包含窗口边框、阴影、透明度和显示缩放 |
| 图像另存为... | 按原始像素尺寸保存当前图像，格式遵循保存对话框选择 |
| OCR 识别 | 识别当前图像并将文字复制到剪贴板；系统 OCR 不可用时显示简短原因 |
| 窗口阴影 | 当前钉住窗口的勾选开关，初始值来自设置 |
| 始终置顶 | 当前钉住窗口的勾选开关，初始值来自设置 |
| 不透明度 | 子菜单：25%、50%、75%、90%、100% |
| 缩放 | 子菜单：25%、50%、75%、100%、125%、150%、200%、适合屏幕、恢复原始大小 |
| 替换图像 | 子菜单：从剪贴板、从文件... |
| 在文件夹中显示 | 仅当图像存在本地源文件或已经保存时显示 |
| 关闭 | 关闭当前钉住窗口，不影响其他钉住窗口 |

第二批增强菜单：

| 菜单项 | 行为 |
| --- | --- |
| 显示工具栏 | 在图片边缘显示紧凑工具栏，提供复制、保存、OCR、旋转和关闭等高频命令 |
| 图像处理 | 子菜单：向左旋转 90°、向右旋转 90°、水平翻转、垂直翻转 |

菜单规则：

- 菜单按“图像输出、窗口显示、图像调整、来源操作、关闭”分组，并使用分隔线。
- 当前生效的阴影、置顶、不透明度和缩放比例应显示勾选状态。
- “替换图像”保留当前窗口位置；默认按新图像原始比例调整窗口尺寸。
- 从剪贴板替换时，剪贴板没有可用图像应显示明确提示，不静默失败。
- 从文件替换只接受应用支持解码的图片格式。
- “在文件夹中显示”不能用于尚未保存的内存截图。
- 菜单底部可显示只读信息：`原始尺寸：宽 × 高`，不做多级尺寸菜单。

以下参考软件菜单不进入当前产品：

- “移动到分组”：需要钉住历史库和分组数据模型，超出当前轻量定位。
- “销毁”：当前没有持久化钉住历史，与“关闭”含义重复且容易误操作。
- 单独的“粘贴”和“替换为文件”：合并为“替换图像”子菜单，减少菜单长度。

## 6. 窗口与界面结构

### 6.1 主窗口

主窗口是截图控制面板，不承担完整设置编辑。

保留：

- 区域截图主操作。
- 当前快捷键摘要。
- 简短状态信息。
- 打开设置。
- 隐藏到托盘。
- 退出程序。

启动时不自动最小化。是否隐藏到托盘由用户主动控制。

### 6.2 设置窗口

设置改为独立 `SettingsWindow`，不再覆盖在主窗口内部。

建议尺寸约为 `420 × 390 px`，根据内容允许小幅调整。界面采用紧凑桌面工具样式：

- Windows 默认字体优先使用 Microsoft YaHei UI，正文约 12–13 px。
- 控件高度约 24 px，设置行高约 28 px。
- 内容边距约 10 px。
- 1 px 中性灰边框。
- 圆角控制在 0–2 px。
- 不使用嵌套卡片、大片阴影和大标题。
- 浅色、深色主题都应保持清晰对比。

顶部固定五个标签：

`常规｜截图｜钉住｜快捷键｜关于`

设置编辑采用草稿模型：

1. 打开窗口时，从当前配置复制一份草稿。
2. “应用”保存并立即应用可热更新的设置。
3. “确定”保存后关闭。
4. “取消”丢弃未保存修改。
5. “恢复默认”恢复当前标签页的默认值，不直接影响其他标签页。

### 6.3 截图遮罩窗口

- 覆盖虚拟桌面。
- 无系统标题栏和窗口边框。
- 未选择区域保持半透明遮罩。
- 鼠标拖动形成的选区恢复原始画面，不改变整个遮罩透明度。
- 选区边缘提供可调整大小的命中区域，但不显示突兀的黑色控制块。
- 工具条优先显示在选区下方，并与选区边框保留间距；空间不足时显示在选区上方。
- 提示信息不得被窗口或屏幕边缘裁切。

### 6.4 钉住窗口

- 无系统标题栏和关闭按钮。
- 无黑色侧边控制块。
- 1 px 边框。
- 默认置顶。
- 按原始宽高比缩放。
- 左键拖动。
- 滚轮缩放。
- 双击关闭。
- 右键打开图片上下文菜单。
- 透明度按设置值初始化。

窗口显示状态与图像数据分离：

- 缩放只调整窗口尺寸，不重采样原图。
- 透明度、阴影、边框和置顶不写入图片。
- 旋转、翻转和替换属于图像数据操作，会影响后续复制、保存和 OCR。
- 每个钉住窗口可临时覆盖阴影、置顶、不透明度和缩放；新窗口仍使用全局设置默认值。

### 6.5 系统托盘

托盘保持常驻，菜单至少包含：

- 区域截图
- 显示控制面板
- 设置
- 隐藏控制面板
- 退出

左键点击托盘图标直接恢复主窗口。主窗口关闭按钮默认隐藏到托盘，不结束进程。

## 8. OCR 设计

### 8.1 基础 OCR

基础 OCR 使用操作系统自带能力，不随应用打包大型模型，保持安装包轻量并支持离线使用。

| 平台 | OCR 实现 | 状态 |
| --- | --- | --- |
| Windows | `Windows.Media.Ocr` | 待实现 |
| macOS | Apple Vision | 规划中 |
| 其他平台 | 无 | 不做 |

基础 OCR 流程：

1. 用户完成区域选择。
2. 点击工具条 OCR 图标，提示为“识别选区文字”。
3. OCR 读取未叠加画笔、矩形和箭头的原始选区图像。
4. 识别成功后，将纯文本复制到系统剪贴板。
5. 主界面或截图状态区域显示结果。

统一状态文案：

- `已识别并复制文字`
- `未识别到文字`
- `缺少中文 OCR 语言包`
- `当前平台不支持系统 OCR`

首个版本不提供 OCR 设置页、语言选择器、结果编辑器和历史记录。语言优先使用操作系统和用户已安装的语言包。

如果当前平台没有可用的系统 OCR，实现可隐藏 OCR 按钮，不显示无效入口。

### 8.2 专业 OCR

以下能力属于规划中的专业版，不计入当前开发承诺：

- 可选下载 PaddleOCR/ONNX 模型。
- 更高中文识别精度。
- 自动语言检测。
- 竖排文字和复杂版面。
- OCR 结果编辑。
- 批量 OCR。
- 表格、Markdown 等结构化导出。

专业模型应独立下载，不直接扩大基础安装包。

### 8.3 OCR 接口

共享业务层只依赖统一接口，不直接调用 Windows 或 macOS API：

```rust
pub trait OcrEngine {
    fn is_available(&self) -> bool;
    fn recognize(&self, image: &CapturedImage) -> anyhow::Result<OcrResult>;
}
```

具体类型和异步方式可在实现阶段调整，但平台能力不得直接散落在 Slint 回调中。

## 9. 目标配置结构

当前配置只包含快捷键和早期 GridStart 字段。下一阶段应迁移到截图工具配置结构，并对旧字段做一次明确迁移或删除，不保留长期双写。

目标结构示例：

```json
{
  "appearance": "system",
  "launch_at_startup": false,
  "hotkey": null,
  "capture": {
    "completion_action": "copy",
    "format": "png",
    "jpeg_quality": 90,
    "save_directory": "",
    "filename_template": "Screenshot_{yyyy-MM-dd_HH-mm-ss}",
    "auto_save": false,
    "save_notification": true
  },
  "pin": {
    "default_opacity": 100,
    "shadow": true,
    "always_on_top": true,
    "wheel_zoom": true,
    "zoom_step": 15,
    "double_click_close": true
  }
}
```

约束：

- 文件编码为 UTF-8。
- 写入采用临时文件加原子替换，避免中途失败破坏现有配置。
- 数值在反序列化后进行范围校验。
- 无效配置应返回明确错误并保留原文件，不静默覆盖。
- 配置保存路径在品牌名确定后需要统一迁移，避免长期使用 `GridStart`。

## 10. 目标架构

Slint 负责窗口、布局、控件状态和用户输入；Rust 负责截图会话、配置、输出、OCR、托盘命令和平台原生能力。

建议目录：

```text
src/
  app/
    controller.rs
    capture_session.rs
    settings.rs
  domain/
    annotation.rs
    output.rs
    ocr.rs
    config.rs
  platform/
    mod.rs
    windows/
      capture.rs
      clipboard.rs
      hotkey.rs
      ocr.rs
      window.rs
    macos/
      capture.rs
      clipboard.rs
      hotkey.rs
      ocr.rs
      window.rs
ui/
  main-window.slint
  settings-window.slint
  overlay-window.slint
  pin-window.slint
  tray.slint
```

平台服务至少包含：

- `CaptureService`
- `ClipboardService`
- `HotkeyService`
- `OcrEngine`
- `WindowService`
- `StartupService`

UI 不直接持有 Win32 或 macOS 句柄。跨线程结果通过 Slint 事件循环回到 UI 线程更新，避免窗口恢复后局部不刷新的问题。

macOS 截图必须处理“屏幕录制”权限；权限缺失时给出明确提示，不返回空白截图。

## 11. 免费版与专业版边界

### 11.1 基础免费功能

- 区域截图。
- 画笔、矩形和箭头标注。
- 撤销与重做。
- 复制和保存。
- 钉住。
- 托盘操作。
- 一个区域截图全局快捷键。
- Windows/macOS 系统 OCR。
- 浅色、深色和跟随系统主题。

### 11.2 规划中的专业功能

- 高精度本地 OCR 模型。
- 复杂版面、表格和批量 OCR。
- OCR 结果编辑和结构化导出。
- 后续确认的商业服务能力。

基础截图流程不得因未登录、无网络或未购买专业版而受限。

## 13. 验收标准

### 13.1 截图

- 多显示器和不同 DPI 下，选区坐标正确。
- 拖动时只有选区恢复原始画面，外层遮罩保持稳定。
- 松开鼠标后选区不跳动、不放大、不改变比例。
- 选区移动和缩放后，复制结果与可见内容一致。
- 标注只在按住左键期间绘制，松开后立即结束。
- 工具条不遮挡选区边框，屏幕边缘处不被裁切。
- 所有工具提示为中文。

### 13.2 钉住

- 初始位置、尺寸和截图选区一致。
- 没有标题栏、黑色关闭按钮和侧边黑块。
- 缩放保持宽高比。
- 拖动、透明度、置顶和双击关闭行为符合设置。
- 可同时创建多个钉住窗口，并继续从托盘或主窗口发起新截图。
- 右键打开上下文菜单，不再直接循环透明度。
- 复制和另存为不包含窗口边框、阴影、透明度及显示缩放。
- “恢复原始大小”准确恢复截图像素尺寸。
- “适合屏幕”保持比例并完整显示，不超出当前显示器工作区。
- 替换图像后窗口位置不变，尺寸按新图像比例更新。
- 无本地源文件时不显示“在文件夹中显示”。

### 13.3 主窗口与托盘

- 用户主动隐藏后，可通过托盘左键和菜单恢复。
- 恢复窗口后完整重绘，无局部空白。
- 启动时不自动最小化。
- 关闭主窗口不退出进程，托盘“退出”才结束程序。

### 13.4 设置

- 取消不修改当前配置。
- 应用后无需关闭窗口即可看到可热更新设置的效果。
- 外观三种模式均可切换，跟随系统能响应系统主题。
- 配置写入失败时保留旧配置并显示明确错误。
- 手动修改配置文件后，提示需要重启应用。

### 13.5 快捷键

- 默认不注册任何全局快捷键。
- 合法且未占用的组合键可触发区域截图。
- 清空后立即注销原快捷键。
- 失败图标和提示原因一致，不显示冗长错误。

### 13.6 OCR

- OCR 使用无标注的原始截图。
- 成功时复制纯文本。
- 无文字、缺少语言包和平台不支持时给出对应状态。
- OCR 处理不得阻塞 Slint UI 线程。

### 13.7 工程质量

- `cargo fmt --check` 通过。
- `cargo test` 通过。
- `cargo clippy` 无新增高优先级问题。
- Release 构建通过。
- Windows 中文源码和文档严格使用 UTF-8。
- 浅色和深色主题下进行截图级视觉检查。

## 14. 明确不做

当前产品不包含：

- 替换系统开始菜单。
- 云同步、云笔记、相册和附件管理。
- 账号密码托管。
- Android、iOS、Web 和 Linux 客户端。
- 鼠标穿透。
- 钉住历史库、分组管理和“销毁”语义。
- 多组全局快捷键。
- 免费版内置大型 OCR 模型。
- 截图遮罩颜色、选区边框和锚点的高级外观设置。
- 未实现的会员、购买、登录和更新入口。

这些内容如需重新进入产品范围，必须单独评估，不直接混入截图工具主线。
